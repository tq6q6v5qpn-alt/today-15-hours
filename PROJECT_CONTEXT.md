# 프로젝트 맥락과 이어가기

마지막 정리: 2026-07-23 (Asia/Seoul)

## 핵심 생각

이 앱의 기준은 언제나 **24시간**이다. 수면 9시간을 보호하면 15시간이 남고, 사용자가 활동을 고를 때마다 준비·이동·활동시간이 그 총량에서 실제로 줄어든다.

시간을 단순히 흘러가는 것으로 보기보다, 지금 여기의 사건과 선택을 지나가는 좌표로 본다. 달력은 달성률 표가 아니라 계획한 사건, 실제 경로, 기억할 선택을 연결하는 좌표 지도다.

## 현재 구현

### 시간 예산

- 수면시간 설정
- 24시간에서 수면과 계획시간 차감
- 남은 시간을 상단에 항상 표시
- 계획이 가용시간을 넘으면 충돌 표시

### 활동

- 빠른 활동: 수영, 독서, 게임·만화·영화, 요리, 베이스, 밴드 활동
- 활동시간을 10분·1시간 단위로 증감
- 준비시간과 이동 전체 시간을 별도로 계획
- 직접 활동 추가

### 오늘 순서

- 길게 눌러 순서 이동
- 위·아래 버튼으로 접근 가능한 순서 이동
- 시작 시각 직접 입력 및 자동 시각 복귀
- 준비·이동·활동을 포함한 전체 소요시간 표시

### 실제 경로

- 준비 시작
- 출발
- 도착
- 활동 시작
- 활동 끝
- 귀가·다음 장소 도착
- 마지막 기록 취소

### 달력

- 날짜별 계획 저장 및 이동
- 선택 사항인 `내가 한 선택` 한 줄
- 생일·기념일·고정사항 추가 및 삭제
- 매년 반복 표시
- 월별 계획시간, 실제 기록시간, 준비·이동시간, 이름 없는 시간 분석

### 데이터

- 날짜별 계획: `today15_calendar_v3`
- 고정사항: `today15_meta_v4`
- 테스트 메모: `today15_feedback_v1`
- 전체 JSON 내보내기와 가져오기
- 기존 단일 날짜 저장 형식 자동 이전

## 데이터 안전성

GitHub에는 앱 코드만 저장된다. 가족 구성원의 계획과 실제 기록은 각 iPhone 브라우저 내부에 따로 저장되며 GitHub로 전송되지 않는다.

브라우저 데이터 삭제, 홈 화면 앱 제거, Safari 설정 초기화 시 기록이 사라질 수 있으므로 주기적으로 JSON 백업을 iCloud Drive 등에 보관한다.

## 현재 기술적 한계

- 정적 GitHub Pages 앱은 완전히 종료된 뒤 자체 정시 알림을 안정적으로 보낼 수 없다.
- 현재는 앱을 열어둔 동안의 알림 또는 Apple 캘린더 `.ics` 알림을 사용한다.
- 백그라운드 위치, Core Motion, Siri/App Intents, Live Activity, HealthKit의 본격 사용에는 네이티브 iOS 앱이 적합하다.
- 가족 간 데이터 자동 동기화는 없다.

## 장기 방향

최종 목표는 Xcode와 SwiftUI 기반 네이티브 iPhone 앱이다.

1. 현재 웹앱으로 가족·지인 실사용 데이터를 모은다.
2. JSON 스키마를 안정화한다.
3. Mac이 준비되면 별도 `today-15-hours-ios` 저장소를 만든다.
4. SwiftData 기반 네이티브 24시간 예산과 JSON 가져오기를 먼저 구현한다.
5. 자체 알림, Siri 음성 기록, 위치·모션 제안, Live Activity, HealthKit 순으로 확장한다.
6. 실제 기록이 쌓이면 활동별 준비·이동·활동시간의 중앙값과 범위를 계산한다.
7. 예: `수영 60분`이 아니라 최근 기록을 근거로 `오늘에서 약 2시간 30분 필요`를 제안한다.
8. 자동 감지는 사용자의 확인 후에만 확정 기록으로 저장한다.
9. 가족 테스트는 TestFlight, 최종 단계는 App Store 배포를 목표로 한다.

5. 실사용 결과를 바탕으로 입력 단계를 줄이거나 자동 제안 후보를 정한다.
6. `core.js`와 `DATA_CONTRACT.md`를 iOS·Android 네이티브 구현의 기준으로 유지한다.
