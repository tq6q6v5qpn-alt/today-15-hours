# iOS·Android 공통 데이터 계약

이 문서는 현재 웹앱과 향후 Swift·Kotlin 앱이 동일하게 해석해야 할 최소 데이터 규칙이다.

## 불변식

1. 하루 총량은 항상 1,440분이다.
2. 가용시간은 `1,440 - 수면시간`이다.
3. 활동의 계획 전체시간은 `준비 + 이동 전체 + 활동`이다.
4. 남은 시간은 음수가 될 수 있으며, 이를 자동으로 잘라내지 않는다.
5. 계획과 실제 사건 기록은 서로 덮어쓰지 않는다.
6. 자동 감지 데이터는 사용자 확인 전까지 확정 기록이 아니다.
7. 기록되지 않은 시간은 낭비로 분류하지 않는다.

## 백업 봉투

```json
{
  "format": "today-15-hours",
  "schemaVersion": 1,
  "exportedAt": "2026-07-23T00:00:00.000Z",
  "calendar": {},
  "meta": { "fixed": [] },
  "feedback": ""
}
```

## 날짜

- 날짜 키: 현지 달력 기준 `YYYY-MM-DD`
- 실제 사건 시각: 시간대가 포함된 ISO 8601 문자열
- 표시 시간대가 바뀌어도 원래 사건 시각과 오프셋을 보존해야 한다.

## 활동

```json
{
  "id": "고유 ID",
  "name": "수영",
  "min": 60,
  "prepMin": 20,
  "travelMin": 60,
  "start": "14:00",
  "focus": {
    "question": "",
    "first": "",
    "finish": ""
  },
  "logs": {
    "prep": "ISO 8601",
    "depart": "ISO 8601",
    "arrive": "ISO 8601",
    "start": "ISO 8601",
    "end": "ISO 8601",
    "return": "ISO 8601"
  }
}
```

`travelMin`은 계획 단계에서 왕복 또는 전체 이동에 확보한 시간이다. 실제 이동은 `depart → arrive`와 `end → return`을 별도로 계산한다.

## 병합

- 서로 다른 날짜는 합친다.
- 같은 날짜는 사용자의 확인 후 가져온 백업으로 교체한다.
- 고정사항은 `id`로 중복 제거한다.
- 알 수 없는 새 필드는 삭제하지 않는 방향으로 마이그레이션한다.
